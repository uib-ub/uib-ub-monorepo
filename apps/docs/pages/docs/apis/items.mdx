## get__items_\{item_id\}

> Code samples

```javascript
const headers = {
  'Accept':'application/ld+json'
};

fetch('https://api-ub.vercel.app/items/{item_id}',
  {
    method: 'GET',
    headers: headers
  })
  .then(function(res) {
      return res.json();
    })
  .then(function(body) {
    console.log(body);
  }
);

```

`GET /items/{item_id}`

*A single object metadata JSON or a IIIF manifest by the given identifier, with the option to as for IIIF or RDF. \{id\} is the identifier from our collections management system.*

### Parameters

| Name    | In    | Type   | Required | Description                                                                                |
| ------- | ----- | ------ | -------- | ------------------------------------------------------------------------------------------ |
| item_id | path  | string | true     | The ID of the manifest to retrieve                                                         |
| as      | query | string | false    | The data domain model to use. Supports 'iiif' and 'rdf'.                                   |
| format  | query | string | false    | The data format to use. Supports 'json' (default), 'turtle' and 'xml'. Ignored if as=iiif. |

> Example responses

> 404 Response

```json
"string"
```

### Responses

| Status | Meaning                                                        | Description                | Schema |
| ------ | -------------------------------------------------------------- | -------------------------- | ------ |
| 200    | [OK](https://tools.ietf.org/html/rfc7231#section-6.3.1)        | Request was successful     | None   |
| 404    | [Not Found](https://tools.ietf.org/html/rfc7231#section-6.5.4) | Requested ID was not found | string |

<aside>
This operation does not require authentication
</aside>
